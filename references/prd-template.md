# PRD And Project Templates

## Project Brief

Use this when the project is early or evidence is incomplete.

```markdown
# <项目名> 项目 Brief

## 背景
## 用户与场景
## 问题定义
## 目标与非目标
## 已有证据
## 机会点
## MVP 建议
## 风险与假设
## 待确认事项
## 下一步
```

## PRD

Use this when the user asks for a PRD or has enough source material.

```markdown
# <项目名> PRD

## 1. 背景
## 2. 目标
## 3. 用户与场景
## 4. 问题与机会
## 5. 范围
## 6. 用户故事
## 7. 功能需求
## 8. 非功能需求
## 9. 数据指标
## 10. 风险与依赖
## 11. 里程碑与排期建议
## 12. 待确认事项
```

## Writing Rules

- Preserve source links.
- Mark unsupported claims as `needs confirmation`.
- Keep scope decisions separate from generated suggestions.
- Put execution tasks in the task plan table, not only in the PRD body.
